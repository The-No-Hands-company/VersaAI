# Smart Model Router & Ensemble - Quick Guide

## 📋 Overview

VersaAI now includes a **Smart Model Router** that automatically selects the best model for each task, and a **Model Ensemble** that can use multiple models simultaneously for consensus-based code generation.

## 🎯 Key Features

### 1. **Automatic Model Selection**
- Analyzes task complexity and requirements
- Detects programming language
- Selects optimal model (fast vs. quality)
- Considers available system resources

### 2. **Multi-Model Ensemble**
- Run multiple models in parallel
- Consensus voting for critical tasks
- Fallback chains for reliability
- Performance tracking

### 3. **Resource-Aware**
- Respects RAM constraints
- Efficient model loading/unloading
- On-demand model initialization

## 🚀 Quick Start

### 1. Download Models

```bash
# List available models
python scripts/download_models.py list

# Check system requirements
python scripts/download_models.py check

# Download specific model (recommended: start with deepseek)
python scripts/download_models.py download deepseek

# Download all models (~26GB)
python scripts/download_models.py download-all
```

### 2. Use Model Router (Python)

```python
from versaai.models.model_router import ModelRouter

# Initialize router with your available RAM
router = ModelRouter(available_ram_gb=16)

# Automatic selection
model_id, model_spec = router.select_model(
    task="Create a Python function for binary search",
    language="python"
)

print(f"Selected: {model_spec.name}")
# Output: Selected: Phi-2 (simple task)

# For complex tasks
model_id, model_spec = router.select_model(
    task="Refactor entire API to use async/await patterns",
    language="javascript"
)

print(f"Selected: {model_spec.name}")
# Output: Selected: WizardCoder-15B (complex refactoring)
```

### 3. Use Model Ensemble

```python
from versaai.models.model_router import ModelRouter
from versaai.models.model_ensemble import ModelEnsemble

# Setup
router = ModelRouter(available_ram_gb=16)

def model_loader(model_id):
    # Load actual model here
    # For now, return mock
    return {"id": model_id}

ensemble = ModelEnsemble(router, model_loader)

# Auto mode - Router selects best model
result = ensemble.generate(
    task="Create a binary search function in Python",
    mode="auto"
)

# Fast mode - Use fastest model
result = ensemble.generate(
    task="Quick hello world script",
    mode="fast"
)

# Quality mode - Use best quality model
result = ensemble.generate(
    task="Implement complex sorting algorithm with optimizations",
    mode="quality"
)

# Consensus mode - Use multiple models and combine results
result = ensemble.generate(
    task="Security-critical authentication system",
    mode="consensus"
)

print(result['code'])
print(f"Generated by: {result['model_name']}")
if 'consensus' in result:
    print(f"Used {result['consensus']['num_models']} models")
```

## 🔧 Advanced Usage

### Custom Model Preferences

```python
# Prefer speed over quality
model_id, spec = router.select_model(
    task="Generate boilerplate code",
    prefer_speed=True

# Prefer quality over speed
```

### Check Available Models

```python
# List all models and their status
models = router.list_models()
for model in models:
    print(f"{model['name']}: {model['tier']} tier")
    print(f"  Languages: {', '.join(model['languages'])}")
    print(f"  Available: {model['available']}")
```

### Performance Statistics

```python
# Get ensemble statistics
stats = ensemble.get_stats()

print("Loaded models:", stats['loaded_models'])

for model_id, s in stats['model_stats'].items():
    if s['calls'] > 0:
        print(f"{model_id}:")
        print(f"  Calls: {s['calls']}")
        print(f"  Success rate: {s['successes']/s['calls']*100:.1f}%")
        print(f"  Avg time: {s['avg_time']:.2f}s")
```

### Memory Management

```python
# Unload specific model to free RAM
ensemble.unload_model("codellama")

# Unload all models
ensemble.unload_all()
```

## 📊 Consensus Mode Details

When using `mode="consensus"`, the ensemble:

1. **Selects 2-3 diverse models** (different sizes/specialties)
2. **Runs them in parallel** using ThreadPoolExecutor
3. **Combines results** intelligently:
   - If all agree (>80% similar) → Use consensus
   - If different → Use result from best model
   - Always include all results for review

**Best for:**
- Production-critical code
- Security-sensitive features
- When quality matters most
- Financial/medical applications

**Example:**
```python
result = ensemble.generate(
    task="Implement encryption system for user passwords",
    mode="consensus"
)

# Access all model outputs
for output in result['consensus']['all_results']:
    print(f"{output['model']}:\n{output['code']}\n")
```

## 🎓 Model Selection Algorithm

The router scores each model based on:

1. **Task Complexity Match** (20 points)
   - Simple → Fast model
   - Medium → Balanced model
   - Complex → Powerful model

2. **Language Specialization** (15 points)
   - Exact match gets +15
   - Multi-language models get +10

3. **Task-Specific Strengths** (10 points each)
   - Debugging → WizardCoder
   - Refactoring → WizardCoder
   - Algorithms → CodeLlama
   - Quick tasks → Phi-2

4. **User Preferences** (15 points)
   - `prefer_speed=True` → Boost fast models
   - `prefer_quality=True` → Boost powerful models

**Highest scoring model is selected!**
