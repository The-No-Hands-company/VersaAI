Components Summary:
------------------

1. Memory Pool Management (VersaAIMemoryPool.hpp)
   Purpose:  High-performance object pooling
   Features: - ObjectPool<T> template
             - 10x faster than new/delete
             - Leak detection
             - RAII support with PoolDeleter
             - Thread-safe operations
             - Growth policies
             - Detailed statistics

2. Dependency Injection (VersaAIDependencyInjection.hpp)
   Purpose:  Type-safe component wiring
   Features: - ServiceCollection (builder)
             - ServiceProvider (resolver)
             - Three lifetimes (Singleton, Transient, Scoped)
             - Circular dependency detection
             - Factory-based construction
             - Scoped services
             - Thread-safe resolution

3. Component Registry (VersaAIRegistry.hpp)
   Purpose:  Lifecycle-aware component management
   Features: - ComponentRegistry (singleton)
             - IInitializable/IShutdownable/IHotReloadable interfaces
             - Priority-based initialization
             - State tracking
             - Metadata (access counts, failures, etc.)
             - DI integration
             - Error recovery
